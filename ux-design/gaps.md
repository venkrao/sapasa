### Role fit & framing
- Walk me through how you’d explain SaPaSa to a **first‑lesson Carnatic student** vs a **teacher** in five minutes each.
- Where should UX sit between **practice tool**, **pedagogy**, and **analytics**—and how would you prioritize?

### Users & journeys
- Who are the **top three personas** you’d design for here (including edge cases like home practice vs noisy rooms)?
- Map the **happy path** from install → first confident session → weekly habit. Where do people drop off?

### Multimodal complexity (audio + pitch + optional camera)
- How do you prevent **speaker bleed**, **wrong octave**, or **disconnected backend** from feeling like “the app is broken” vs “something I can fix”?
- When live pitch disagrees with what the singer **feels**, how should the UI respond (trust, correction, education)?

### Melody Capture (standalone module)
- How would you validate whether **capture modes** (raw / piano‑friendly / aggressive / anti‑octave) help or confuse—and how would you **name** them for students?
- The replay timeline is **musical**, not wall‑clock capture length—how would you make that obvious without clutter?

### Carnatic Training & exercises
- Exercise tempo, expected swara, holds, reference tone—how do you balance **guidance** vs **cognitive load** on one screen?
- How would you design **progressive disclosure** so beginners aren’t overwhelmed by tanpura, graph, exercises, camera, timer at once?

### Trust, explanation, and errors
- What **microcopy + patterns** would you use for states like disconnected, empty capture, analysis with low confidence?
- Where would you put **“why am I seeing this?”** explanations (tooltips vs coach marks vs help panel)?

### Accessibility & inclusivity
- How would you test **color‑only pitch feedback** for accessibility—and what non‑color channels would you add?
- Any considerations for **older learners**, **non‑English UI**, or **low‑tech environments**?

### Research & iteration
- What **three studies** (moderated usability, diary study, A/B, etc.) would you run first—and what decisions would each unblock?
- Pick one controversial UX tradeoff we’ve implicitly made (e.g. aggressive melody segmentation vs fidelity). How would you decide?

### Collaboration
- How would you work with **engineering** when constraints are real‑time audio, ML/heuristics, and browser APIs?
- How would you involve **a Carnatic teacher** in the design process without letting expert bias erase beginners?

### Portfolio tie‑in
- Show something where you improved **trust in ambiguous automated feedback**—what did you measure before/after?

These are intentionally tied to SaPaSa’s real tensions: **technical honesty**, **musical culture**, and **learner emotion**.
